# Remaining Functional Tasks

## 1. 하드웨어 연동 고도화
- MQTT/WebSocket 기반 실시간 기기 제어/센서 스트림으로 전환(현재는 HTTP + 큐 기반 스텁)
- 기기 ACK 응답 패턴 정의 및 재시도/타임아웃 관리
- 실제 RTSP 스트림 캡처 서비스 연동 및 이미지 저장소(S3 등) 결정

## 2. 캐릭터 · 리포트 세부 로직
- 캐릭터 상태 룰 엔진 구현(센서 지표 기반 감정/메시지 매칭)
- 환경경 점수 공식/가중치 확정 및 관리자 변경 API
- 시계열 집계 결과를 기반으로 한 대시보드 차트 응답 포맷 정교화

## 3. 알림/설정 & 사용자 경험
- FCM/이메일/SMS 등 외부 알림 채널 연동
- 방해 금지 모드, 알림 우선순위, 사용자별 큐 스로틀링 구현
- 빠른 명령 및 사용자 설정 초기 데이터 seeding

## 4. 채팅 & 명령 플로우 고도화
- 자연어 명령 파싱(의도 분류 → 기기 제어/리포트 질의 연계)
- 채팅 히스토리 무한 스크롤/페이지네이션 API
- 관리자용 시스템 메시지 편집 기능

## 5. 테스트 & 모니터링
- 단위/통합/큐 워커 테스트 확대 (Jest + Testcontainers 등)
- Metrics/Health 체크(큐 지표, DB 연결, Redis 상태) API
- 로깅/트레이싱(Winston 연동, OpenTelemetry 고려)

## 6. 배포 및 운영
- Docker/Docker Compose 및 PM2 ecosystem 설정 정리
- `.env.example` 갱신 및 환경별 구성 문서화
- CI/CD 파이프라인(빌드, 테스트, 배포) 정의
